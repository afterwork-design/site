                    <h1 id="直接建模 vs 基于历史的参数化建模">
                        <a href="#直接建模 vs 基于历史的参数化建模">直接建模 vs 基于历史的参数化建模</a>
                    </h1>
                <p>三维建模软件调研笔记。</p>

                    <h2 id="直接建模（Direct Modeling）">
                        <a href="#直接建模（Direct Modeling）">直接建模（Direct Modeling）</a>
                    </h2>
                <p>直接建模的最大优势就是<strong>所见即所得</strong>。</p>

                    <h3 id="概念">
                        <a href="#概念">概念</a>
                    </h3>
                <p>直接建模是一种无需依赖于操作顺序和参数关系的三维建模方法，用户直接对几何体的点线面进行编辑和操作。</p>
<p>直接建模提供了更大的灵活性和直观性，使得对模型的编辑和修改更为简便快捷，更适合于快速概念验证、复杂形状编辑以及处理无历史数据的现有模型。</p>

                    <h3 id="应用">
                        <a href="#应用">应用</a>
                    </h3>
                <p><strong>SketchUp</strong></p>
<p>SketchUp 又被称为草图大师，是建筑领域常用的概念设计软件，</p>
<p><img src="../assests/blog/image/sketchup.png" alt="SketchUp"></p>
<p><strong>Shapr3D</strong></p>
<blockquote>
<p><a href="https://www.shapr3d.com/">www.shapr3d.com</a></p>
</blockquote>
<p>Shapr3D 最初是在 iPad 上使用的建模软件，所以在交互上设计得非常直观。</p>
<p>在 2024 年，Shapr3D 也推出了基于历史的参数化建模版本，并且保留了直接建模的交互方式。</p>
<p><img src="../assests/blog/image/sharp3d-history.png" alt="Shapr3D history"></p>
<ul>
<li>SolidWorks：是一款领先的 CAD 软件，提供了直接建模工具，允许用户在不依赖历史记录的情况下编辑模型。</li>
<li>Autodesk Fusion 360：结合了参数化和直接建模的特点，提供了灵活的设计环境，适合产品开发和机械设计。</li>
<li>Rhino</li>
<li>CAXA实体设计（IRONCAD)</li>
<li>Creo Direct</li>
<li>Creo Elements</li>
<li>KeyCreato</li>
</ul>

                    <h3 id="建模流程">
                        <a href="#建模流程">建模流程</a>
                    </h3>
                <ol>
<li>绘制轮廓：使用线、圆、矩形等工具创建 2D 轮廓</li>
<li>创建 3D 模型：通过推、拉、布尔等操作创建 3D 模型</li>
<li>修改模型：直接编辑几何体</li>
</ol>

                    <h3 id="局限性">
                        <a href="#局限性">局限性</a>
                    </h3>
                <ul>
<li>参数化控制不足：直接建模可能缺乏参数化建模中的精确尺寸和关系控制，导致设计迭代和修改时的不便。</li>
<li>历史记录缺失：直接编辑可能会导致设计意图的丢失，使得追踪变更和协作变得更加困难。</li>
<li>功能限制：某些高级功能，如复杂装配和详细工程图的生成，可能在直接建模环境中不如参数化建模环境完善。</li>
</ul>

                    <h2 id="基于历史的参数化建模（History-Based Parametric Modeling）">
                        <a href="#基于历史的参数化建模（History-Based Parametric Modeling）">基于历史的参数化建模（History-Based Parametric Modeling）</a>
                    </h2>
                <p>基于历史的参数化建模的最大优势就是<strong>可追溯</strong>。</p>

                    <h3 id="概念">
                        <a href="#概念">概念</a>
                    </h3>
                <p>参数化建模是一种基于定义几何形状的参数和关系的三维建模方法，通过修改这些参数来控制和更新模型的结构和外观。</p>
<p>参数化建模能够通过调整参数快速进行设计迭代和保持设计的一致性，适用于需要精确控制设计变量和进行大量迭代的设计任务。</p>

                    <h3 id="应用">
                        <a href="#应用">应用</a>
                    </h3>
                <p>支持参数化建模的软件有很多。</p>

                    <h4 id="国内软件">
                        <a href="#国内软件">国内软件</a>
                    </h4>
                <ul>
<li>CrownCAD</li>
</ul>
<p><img src="../assests/blog/image/crowncad.png" alt="CrownCAD"></p>
<ul>
<li>中望 3D</li>
</ul>

                    <h4 id="国外软件">
                        <a href="#国外软件">国外软件</a>
                    </h4>
                <ul>
<li>CATIA：是 Dassault Systemes 开发的专业CAD软件，广泛应用于航空航天、汽车和工业设计领域，以其强大的参数化建模功能而知名。</li>
<li>NX：由西门子PLM软件开发，是一款集成了多种设计和制造功能的高级CAD软件，支持基于历史的参数化建模。</li>
<li>Creo Parametric：是PTC公司开发的一款CAD软件，继承了Pro/ENGINEER的特点，专注于基于历史的参数化设计。</li>
<li>Solidworks</li>
<li>Autodesk Inventor</li>
<li>Fusion360</li>
<li>Onshape</li>
</ul>

                    <h3 id="建模流程">
                        <a href="#建模流程">建模流程</a>
                    </h3>
                <p>大多数参数化建模软件的操作流程都包含以下几个主要步骤：</p>
<ol>
<li>创建二维草图：在二维平面里绘制形状，创建模型的基础2D轮廓。</li>
<li>设置参数和关系：在草图中定义轮廓尺寸和关系，即创建尺寸约束和几何约束。这些关系将控制模型的几何形状能够根据参数的变化而自动更新。</li>
<li>生成三维模型：通过使用拉伸、旋转等特征操作，将二维草图转换为三维实体模型。在这个过程中，模型树区域会记录所有的尺寸和参数信息，允许设计师随时进行修改和更新。</li>
<li>修改模型：定位到对应的特征节点去修改</li>
</ol>
<p>与直接建模最大区别在于修改模型，参数化建模会记录历史操作，当需要修改模型时，可以定位到对应的特征节点去修改。</p>

                    <h3 id="局限性">
                        <a href="#局限性">局限性</a>
                    </h3>
                <ul>
<li>复杂性：对于初学者来说，参数化建模的学习曲线可能比较陡峭，需要时间来理解和掌握。</li>
<li>性能问题：大型的参数化模型可能会因为复杂的历史记录和关联关系而导致软件性能下降。</li>
<li>过度约束：过多的参数和关系可能导致模型过于复杂和脆弱，轻微的更改可能会引起连锁反应，导致模型崩溃或错误。</li>
</ul>
<p>所以，直接建模与基于历史的参数化建模并不是非此即彼的关系，要根据不同的设计阶段，选用适合的建模方式。</p>
